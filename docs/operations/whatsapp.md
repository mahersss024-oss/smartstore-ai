# WhatsApp Operations

SmartStore AI supports two store-level WhatsApp providers:

- **Meta WhatsApp Cloud API**: direct Meta Cloud API integration.
- **Whapi.cloud**: managed WhatsApp provider integration.

Each store chooses one provider from the dashboard. The conversation engine,
guardrails, catalog, pricing, cart, and order workflows are provider-independent.

## Activation Checklist

Use this checklist for each merchant store before enabling WhatsApp traffic.

### Option A: Meta WhatsApp Cloud API

From the store's Meta app, collect:

- **Phone Number ID**: numeric id of the WhatsApp sender number.
- **Access Token**: use a System User token for production.
- **WhatsApp Business Account ID (WABA)**: optional reference id.
- **Display phone number**: the human-readable sender number, such as `+9665...`.

In the Meta app webhook settings:

```text
Callback URL: https://smartstore-ai.com/api/whatsapp/webhook
Verify token: the value of META_WEBHOOK_VERIFY_TOKEN
```

Subscribe to the `messages` field. The platform must already be deployed with
`META_WEBHOOK_VERIFY_TOKEN` set before verification.

### Option B: Whapi.cloud

Preferred merchant flow:

1. In Render, configure platform Whapi managed-connect variables:
   - `WHAPI_PARTNER_API_TOKEN`
   - `WHAPI_PROJECT_ID`
2. In the store dashboard, select **Whapi.cloud** and click **Show QR**.
3. The platform creates/reuses the Whapi channel, configures the webhook,
   encrypts the channel token, and shows the QR for the merchant to scan.

Manual fallback:

From the store's Whapi account/channel, collect:

- **Channel ID**: the Whapi channel identifier.
- **API Token**: the token used to send messages through Whapi.
- **Display phone number**: the WhatsApp sender number shown to customers.
- **Webhook secret**: generated by the platform if left blank when saving.

After saving the Whapi settings in SmartStore, copy the generated webhook URL
from the dashboard and configure it in Whapi:

```text
https://smartstore-ai.com/api/whatsapp/webhook?provider=whapi&channelId=<channelId>&secret=<webhookSecret>
```

## SmartStore Store Dashboard

```text
Dashboard -> Settings -> WhatsApp / Customer Channels
```

Select the provider, enter that provider's credentials, and save. Provider
secrets are encrypted with AES-256-GCM before storage. Leaving an existing
secret field blank preserves the saved encrypted value.

## Required Platform Environment

Platform-level WhatsApp environment values:

- `PLATFORM_SECRETS_ENCRYPTION_KEY`: encrypts per-store provider tokens.
- `META_APP_SECRET`: required only for Meta webhook signature verification.
- `META_WEBHOOK_VERIFY_TOKEN`: required only for Meta webhook verification.
- `NEXT_PUBLIC_APP_URL`: stable absolute webhook/customer links.

Whapi manual credential entry does not require global Render environment
variables. The direct QR flow requires `WHAPI_PARTNER_API_TOKEN` and
`WHAPI_PROJECT_ID`. Store channel tokens and webhook secrets are stored per store
in `channel_connections.config`.

Optional, for durable async AI worker mode:

- `AI_PROCESSING_MODE=outbox`
- `QSTASH_TOKEN`
- `QSTASH_CURRENT_SIGNING_KEY`
- `QSTASH_NEXT_SIGNING_KEY`
- `CRON_SECRET`

Leave `AI_PROCESSING_MODE` unset (`sync`) until the worker is provisioned.

## Database Requirements

Apply all migrations before production traffic. WhatsApp activation depends on:

- `channel_connections` containing one active `whatsapp` row per organization.
- `channel_connections.config.provider` being either `meta` or `whapi`.
- Provider-specific encrypted token fields being present for the selected
  provider.

## Smoke Test

### Meta

1. Verify the customer phone as a recipient when using Meta test numbers.
2. Send a WhatsApp message from the customer phone to the store's number.
3. Confirm `/api/whatsapp/webhook` logs `Meta WhatsApp webhook received`.
4. Confirm `Meta outbound reply sent` appears and the customer receives it.

### Whapi

1. Use the store dashboard QR flow, or configure the Whapi webhook URL copied
   from the store dashboard.
2. Send a WhatsApp message from the customer phone to the Whapi channel number.
3. Confirm `/api/whatsapp/webhook?provider=whapi...` logs
   `Whapi WhatsApp webhook received`.
4. Confirm `Whapi outbound reply sent` appears and the customer receives it.

## Incoming Messages

The shared route `/api/whatsapp/webhook`:

1. Selects provider mode:
   - Meta: default route, verifies `X-Hub-Signature-256`.
   - Whapi: `provider=whapi`, verifies the store webhook secret.
2. Parses the inbound message.
3. Resolves exactly one active store by provider-specific channel id.
4. Passes the message to the shared conversation engine.
5. Sends the reply through the selected provider.

## Isolation And Safety

- Each store is resolved by its provider-specific channel id.
- Access tokens and API tokens are encrypted and never returned to the browser.
- Secrets are not written to logs.
- A store without valid provider credentials remains inactive.
- Retryable processing returns HTTP 503 without exposing provider details.
- Conversation, guardrail, catalog, pricing, and order logic are
  provider-independent.
